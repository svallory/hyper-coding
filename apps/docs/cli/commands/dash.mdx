# hyper dash

Launch the interactive Terminal User Interface (TUI) dashboard that provides comprehensive project monitoring, real-time analytics, team collaboration views, and centralized control of your HyperDev workflow.

## Syntax

```bash
hyper dash [view] [options]
```

## Overview

The `hyper dash` command launches a full-featured TUI dashboard that serves as mission control for your project:

- **Real-time monitoring**: Live project metrics, build status, and team activity
- **Interactive navigation**: Keyboard-driven interface for efficient project management
- **Multi-view dashboard**: Switch between different perspectives (dev, team, epic, analytics)
- **Live updates**: Auto-refreshing data with configurable intervals
- **Team collaboration**: Real-time team activity and communication
- **Quick actions**: Perform common tasks directly from the dashboard

<Info>
  The TUI dashboard provides a comprehensive overview of your entire HyperDev project in a single,
  interactive terminal interface. It's designed for developers who prefer keyboard-driven workflows.
</Info>

## Views

| View | Description | Shortcut |
|------|-------------|----------|
| `overview` | Project overview with key metrics | `o` |
| `epics` | Epic management and progress tracking | `e` |
| `tasks` | Task management and assignment | `t` |
| `team` | Team activity and collaboration | `m` |
| `dev` | Development monitoring and quality | `d` |
| `analytics` | Performance analytics and insights | `a` |
| `logs` | System logs and activity feed | `l` |

## Options

| Option | Description | Default | Example |
|--------|-------------|---------|---------|
| `--refresh <seconds>` | Auto-refresh interval | `30` | `--refresh 10` |
| `--theme <theme>` | Dashboard theme | `auto` | `--theme dark` |
| `--layout <layout>` | Dashboard layout | `default` | `--layout minimal` |
| `--focus <view>` | Start with specific view | `overview` | `--focus team` |
| `--port <port>` | Web dashboard port | `3001` | `--port 8080` |
| `--web` | Launch web dashboard | `false` | `--web` |
| `--readonly` | Read-only mode | `false` | `--readonly` |

## Examples

### Basic Dashboard Usage

```bash
# Launch default dashboard
hyper dash

# Start with specific view
hyper dash --focus epics

# Launch with faster refresh rate
hyper dash --refresh 5

# Dark theme with minimal layout
hyper dash --theme dark --layout minimal
```

### Specialized Views

```bash
# Epic-focused dashboard
hyper dash epics

# Team collaboration view
hyper dash team

# Development monitoring
hyper dash dev --refresh 15

# Analytics and reports
hyper dash analytics
```

### Web Dashboard

```bash
# Launch web-based dashboard
hyper dash --web

# Web dashboard on custom port
hyper dash --web --port 8080

# Readonly web dashboard for stakeholders
hyper dash --web --readonly --port 3000
```

## Dashboard Navigation

### Keyboard Shortcuts

#### Global Navigation
- `Tab` / `Shift+Tab` - Switch between views
- `q` / `Ctrl+C` - Quit dashboard  
- `r` - Force refresh current view
- `?` - Show help and shortcuts
- `Ctrl+R` - Restart dashboard

#### View-Specific Shortcuts
- `o` - Switch to Overview
- `e` - Switch to Epics  
- `t` - Switch to Tasks
- `m` - Switch to Team
- `d` - Switch to Development  
- `a` - Switch to Analytics
- `l` - Switch to Logs

#### List Navigation
- `↑` / `↓` - Navigate up/down
- `Enter` - Select/open item
- `Space` - Toggle selection
- `Home` / `End` - Go to first/last item
- `/` - Search/filter

### Quick Actions

#### Epic Management
- `n` - Create new epic
- `u` - Update selected epic
- `a` - Assign team members
- `v` - View epic details
- `s` - Change epic status

#### Task Management  
- `c` - Create new task
- `a` - Assign task
- `s` - Update task status
- `d` - Set due date
- `p` - Set priority

#### Development Actions
- `b` - Trigger build
- `t` - Run tests
- `f` - Fix automatically fixable issues
- `d` - Deploy to staging
- `s` - Security scan

## Dashboard Customization

### Themes

```bash
# Available themes
hyper dash --theme light
hyper dash --theme dark  
hyper dash --theme high-contrast
hyper dash --theme solarized
hyper dash --theme custom
```

### Layouts

```bash
# Layout options
hyper dash --layout default      # Standard 4-panel layout
hyper dash --layout minimal      # Simplified 2-panel layout  
hyper dash --layout wide         # Optimized for wide screens
hyper dash --layout focus        # Single-panel focus mode
hyper dash --layout custom       # User-defined layout
```

### Custom Views

```bash
# Create custom dashboard view
hyper dash config view create my-view \
  --panels "epics,team,recent-activity" \
  --layout grid \
  --refresh 10

# Use custom view
hyper dash my-view

# Save current view as default
hyper dash config save-layout
```

## Web Dashboard

### Web Interface

```bash
# Launch web dashboard
hyper dash --web

# Access at http://localhost:3001
# Features:
# - Responsive design for all devices
# - Real-time updates via WebSocket
# - Shareable URLs for specific views
# - Screenshot and PDF export
# - Team collaboration features
```

### Web Dashboard Features

- **Responsive Design**: Works on desktop, tablet, and mobile
- **Real-time Updates**: Live data updates without refresh
- **Shareable Links**: Share specific views with team members
- **Export Options**: Generate PDF reports and screenshots
- **Collaborative Features**: Comments, annotations, and discussions
- **Embedding**: Embed dashboard views in other tools

### Stakeholder Views

```bash
# Readonly dashboard for stakeholders
hyper dash --web --readonly

# Executive dashboard with high-level metrics
hyper dash --web --view executive

# Client-facing project status  
hyper dash --web --view client --readonly
```

## Dashboard Configuration

### Settings

```javascript
// hyperdev.config.js
export default {
  dashboard: {
    // TUI settings
    tui: {
      theme: 'auto',           // auto, dark, light
      refresh_interval: 30,    // seconds
      default_view: 'overview',
      enable_mouse: true,
      enable_colors: true
    },
    
    // Web dashboard
    web: {
      port: 3001,
      enable_auth: false,
      theme: 'corporate',
      real_time_updates: true,
      export_formats: ['pdf', 'png']
    },
    
    // View configuration
    views: {
      overview: {
        panels: ['health', 'epics', 'activity', 'metrics'],
        layout: 'grid'
      },
      team: {
        show_activity: true,
        show_chat: true,
        show_presence: true
      },
      dev: {
        auto_refresh: 15,
        show_logs: true,
        alert_on_failures: true
      }
    },
    
    // Notifications
    notifications: {
      desktop: true,
      sound: false,
      build_failures: true,
      security_alerts: true,
      epic_updates: false
    }
  }
};
```

## Performance and Resources

### Resource Usage

The TUI dashboard is optimized for minimal resource usage:
- **Memory**: ~50-100MB typical usage
- **CPU**: Less than 5% during normal operation
- **Network**: Minimal data transfer (only updates)
- **Battery**: Efficient for laptop development

### Performance Tips

```bash
# Reduce refresh rate for battery savings
hyper dash --refresh 60

# Use minimal layout for slower systems
hyper dash --layout minimal --refresh 45

# Readonly mode uses fewer resources
hyper dash --readonly
```

- Security alerts are triggered

### Quick Actions
