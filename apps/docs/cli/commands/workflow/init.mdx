# hyper init

Initialize a new HyperDev project with optimal configuration.

## Syntax

```bash
hyper init [options]
```

## Overview

The `hyper init` command creates a new project with everything you need for AI-augmented development:

- Project structure optimized for your stack
- Development environment configuration
- AI provider integration  
- Template system setup
- Quality gates and monitoring
- Team collaboration features

<Info>
  `hyper init` combines functionality from multiple tools ([DX](/tools/dx), [Gen](/tools/gen)) to provide 
  a unified project initialization experience.
</Info>

## Options

| Option | Description | Default | Example |
|--------|-------------|---------|---------|
| `--dry-run` | Preview what would be created | `false` | `--dry-run` |
| `--force` | Overwrite existing files | `false` | `--force` |
| `--verbose` | Detailed output | `false` | `--verbose` |

## Examples

### Basic Initialization

```bash
# Interactive setup (default)
hyper init

# Quick setup with defaults
hyper init --no-interactive
```

The interactive setup guides you through:
1. Development environment setup
2. Basic configuration options



## What Gets Created

### Project Structure

```
my-project/
├── hyperdev.config.js      # Project configuration
├── .hyper/                 # HyperDev workspace
│   ├── templates/          # Local templates
│   ├── cache/              # Template cache
│   ├── epics/              # Project epics
│   └── reports/            # Generated reports
├── package.json            # Dependencies
├── tsconfig.json           # TypeScript config
├── .gitignore              # Git ignore rules
├── README.md               # Project documentation
└── src/                    # Source code
    ├── components/         # Reusable components
    ├── pages/              # Application pages
    ├── api/                # API routes
    ├── lib/                # Utility libraries
    └── tests/              # Test files
```

### Configuration Files

**hyperdev.config.js**
```javascript
export default {
  // AI Configuration
  aiProvider: 'openai',
  aiModel: 'gpt-4',
  
  // Template Configuration
  templateSources: [
    'official',
    'community',
    './local-templates'
  ],
  trustThreshold: 8.0,
  
  // Development Configuration
  qualityGates: {
    security: 'strict',
    performance: 'standard',
    testing: 'required'
  },
  
  // IDE Integration
  ide: {
    primary: 'cursor',
    extensions: ['hyperdev', 'copilot'],
    settings: {
      aiAssistance: true,
      codeGeneration: true
    }
  }
};
```

### Development Environment

The init command sets up:

1. **Package Management**: Configures bun/npm with optimal settings
2. **TypeScript**: Full type safety with strict configuration  
3. **Linting & Formatting**: ESLint, Prettier, and Biome
4. **Testing**: Vitest or Jest with coverage reporting
5. **Git Hooks**: Pre-commit hooks for quality checks
6. **CI/CD**: GitHub Actions or similar workflows
7. **Documentation**: Auto-generated docs and README

## Advanced Usage

### Custom Templates

```bash
# Use local template
hyper init --template ./my-custom-template

# Use remote template
hyper init --template github:username/repo-template

# Use npm template
hyper init --template @company/project-template
```

### Environment-Specific Setup

```bash
# Development environment
hyper init --template saas-app --env development

# Production-ready setup
hyper init --template saas-app --env production --team-config prod.json

# Staging environment
hyper init --template saas-app --env staging
```

### Monorepo Initialization

```bash
# Initialize monorepo structure
hyper init --template monorepo-base

# Add workspace packages
hyper init --template api-package --workspace packages/api
hyper init --template ui-package --workspace packages/ui
```

## Configuration

The `hyper init` command respects HyperDev's configuration hierarchy, applying settings in priority order:

### Configuration Priority (Highest to Lowest)

1. **Command-line flags** - Always take precedence
2. **Environment variables** - `HYPERDEV_*` variables
3. **Project context** - Inherited from parent directories
4. **User configuration** - Your personal defaults
5. **System defaults** - Built-in fallbacks

### Setting Init Defaults

```bash
# Set user-level defaults (affects all new projects)
hyper config set default-template fullstack-saas
hyper config set default-ide cursor  
hyper config set ai-provider openai

# Set environment-specific defaults
export HYPERDEV_DEFAULT_TEMPLATE=minimal
export HYPERDEV_DEFAULT_IDE=vscode

# Command-line flags override all other settings
hyper init --template react-lib --ide cursor --ai-provider anthropic
```

### Configuration Examples

<CodeGroup>
```bash User Defaults
# These settings apply to all new projects
~/.hyperdevrc.js:
{
  defaults: {
    template: "fullstack-saas",
    ide: "cursor",
    aiProvider: "openai"
  }
}
```

```bash Environment Override
# Override user defaults with environment variables
export HYPERDEV_DEFAULT_TEMPLATE=minimal
export HYPERDEV_AI_PROVIDER=anthropic
hyper init  # Uses environment settings
```

```bash Command Override
# Command flags have highest priority
hyper init --template react-lib --ide vscode
# Ignores both user defaults and environment variables
```
</CodeGroup>

<Tip>
**Configuration Tip**: Use `hyper config show` to see your resolved configuration before running `hyper init` to understand which settings will be applied.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Template not found">
    **Problem**: `Template 'my-template' not found`
    
    **Solutions**:
    ```bash
    # List available templates
    hyper list templates
    
    # Search for similar templates
    hyper search "my-template"
    
    # Update template cache
    hyper config clear-cache clear && hyper init
    ```
  </Accordion>
  
  <Accordion title="Permission errors during setup">
    **Problem**: Permission denied when creating files
    
    **Solutions**:
    ```bash
    # Check directory permissions
    ls -la
    
    # Use different directory
    mkdir ~/my-project && cd ~/my-project && hyper init
    
    # Run with elevated permissions (if necessary)
    sudo hyper init --force
    ```
  </Accordion>
  
  <Accordion title="AI provider not configured">
    **Problem**: AI functionality not working after init
    
    **Solutions**:
    ```bash
    # Configure AI provider
    export OPENAI_API_KEY="your-key"
    hyper config set ai-provider openai
    
    # Test AI connection
    hyper generate --test-ai-connection
    ```
  </Accordion>
  
  <Accordion title="IDE integration failing">
    **Problem**: IDE features not working correctly
    
    **Solutions**:
    ```bash
    # Reconfigure IDE
    hyper config --ide cursor
    
    # Check IDE configuration
    hyper config list | grep ide
    
    # Reset IDE settings
    hyper config set default-ide cursor --force
    ```
  </Accordion>
</AccordionGroup>
