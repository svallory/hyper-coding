## Command Patterns

### Consistent Structure
All HyperDev CLI commands follow predictable patterns:

```bash
hyper [command] [subcommand] [options] [arguments]
```

**Examples:**
```bash
hyper init --template fullstack      # Simple workflow command
hyper gen component --name User # Generate with options
hyper plan create --name "Auth"      # Subcommand with arguments
hyper dev watch --focus api          # Advanced tool with options
```

### Common Options

These options work across most commands:

| Option | Description | Example |
|--------|-------------|---------|
| `--help` | Show command help | `hyper gen --help` |
| `--verbose` | Detailed output | `hyper init --verbose` |
| `--dry-run` | Preview without changes | `hyper gen --dry-run` |
| `--json` | JSON format output | `hyper status --json` |
| `--interactive` | Interactive mode | `hyper init --interactive` |

### Help System

Get help at any level:

```bash
# Global help
hyper --help

# Category help
hyper gen --help
hyper plan --help

# Subcommand help
hyper plan create --help
hyper dev watch --help

# Topic help
hyper help workflows
hyper help templates
hyper help configuration
```

## Usage Patterns

### Beginner Workflow
```bash
hyper init                    # Start new project
hyper gen component      # Generate code
hyper plan                    # Plan features
hyper status                  # Check progress
```

### Intermediate Workflow  
```bash
hyper search api   # Find cookbooks and recipes
# suppose an elysia-api cookbook is among the results
hyper gen elysia-api
hyper plan "User Authentication" # Starts an interactive planning session
hyper spec "User Authentication"
hyper tasks from-plan "User Authentication"
hyper dev "User Authentication"
# In another terminal, to follow the progress
hyper dash
```

### Advanced Workflow
```bash
hyper gen search --trust-level 9
hyper gen compose --templates api,ui,tests  
hyper epics arch --diagram system
hyper dev validate --security --performance
hyper dash export --format json --team
```

Commands work together naturally:

```bash
# Complete feature development flow
```

HyperDev provides intelligent error messages with suggestions:

```bash
  - api-template (trust: 8.9)

Try:
  hyper search "unknwon-template"
  hyper list templates --filter component
  hyper gen react-component

Need help? Run: hyper help templates
```

## Configuration Impact

All CLI commands respect HyperDev's hierarchical configuration system, which resolves settings in priority order:

<Steps>
  <Step title="Command-line flags">
    Highest priority - overrides all other settings
  </Step>
  <Step title="Environment variables">
    `HYPERDEV_*` variables override configuration files
  </Step>
  <Step title="Project configuration">
    `hyperdev.config.js` in project root or parent directories
  </Step>
  <Step title="User configuration">
    Personal settings in `~/.hyperdevrc` or `~/.config/hyperdev.config.js`
  </Step>
  <Step title="System defaults">
    Built-in defaults when no other configuration is found
  </Step>
</Steps>

### Configuration Examples

```bash
# Environment variable overrides (highest priority)
HYPERDEV_DEFAULT_IDE=cursor hyper init

# Command-line flag overrides (highest priority)
hyper init --ide vscode     # Overrides environment and config file settings

# Global user settings (lower priority)
hyper config set default-ide cursor
hyper config set ai-provider openai
hyper config set trust-threshold 8

# Project settings (higher priority than user settings)
# Defined in hyperdev.config.js in your project
```

<Info>
**Configuration Precedence**: Command-line flags always win, followed by environment variables, then project config, then user config, and finally system defaults. See the [Configuration Guide](/configuration-guide) for complete details.
</Info>
