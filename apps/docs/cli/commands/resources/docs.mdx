# hyper docs

Manage project documentation including technical specifications, user guides, API documentation, and project knowledge. Create, organize, update, and maintain comprehensive project documentation.

## Syntax

```bash
hyper docs [action] [options]
```

## Overview

The `hyper docs` command provides comprehensive documentation management:

- **Documentation creation**: Generate technical specs, user guides, and API docs
- **Content organization**: Structure and categorize documentation effectively
- **AI-assisted writing**: Get help writing clear, comprehensive documentation
- **Template system**: Use templates for consistent documentation standards
- **Version control**: Track documentation changes and maintain history
- **Integration**: Connect docs with epics, tasks, and code changes

<Info>
  Project documentation includes all written materials that help team members and users understand,
  use, and contribute to your project. This is separate from CLI or code documentation.
</Info>

## Actions

| Action | Description | Example |
|--------|-------------|---------|
| `list` | List all documentation | `hyper docs list` |
| `create` | Create new documentation | `hyper docs create api-guide` |
| `edit` | Edit existing documentation | `hyper docs edit user-manual` |
| `generate` | AI-generate documentation | `hyper docs generate --type api` |
| `organize` | Organize documentation structure | `hyper docs organize` |
| `search` | Search documentation content | `hyper docs search "authentication"` |
| `validate` | Validate documentation quality | `hyper docs validate` |
| `export` | Export documentation | `hyper docs export --format pdf` |
| `serve` | Serve documentation site | `hyper docs serve` |

## Options

| Option | Description | Default | Example |
|--------|-------------|---------|---------|
| `--type <type>` | Documentation type | `guide` | `--type api` |
| `--template <name>` | Template to use | `standard` | `--template technical-spec` |
| `--epic <name>` | Associate with epic | - | `--epic user-auth` |
| `--format <format>` | Output format | `markdown` | `--format pdf` |
| `--ai-assist` | Enable AI assistance | `false` | `--ai-assist` |
| `--interactive, -i` | Interactive mode | `false` | `--interactive` |
| `--output <path>` | Output directory | `docs/` | `--output ./documentation` |

## Examples

### Basic Documentation Management

```bash
# List all project documentation
hyper docs list

# List by category
hyper docs list --type api

# Create new documentation
hyper docs create "API Authentication Guide"

# Edit existing documentation
hyper docs edit user-manual
```

### AI-Assisted Documentation

```bash
# Generate API documentation from code
hyper docs generate --type api --source ./src/api

# Create user guide with AI assistance
hyper docs create "User Onboarding Guide" --ai-assist

# Generate technical specification for epic
hyper docs generate --type technical-spec --epic user-auth

# Interactive documentation creation
hyper docs create --interactive --ai-assist
```

### Documentation Types

```bash
# Technical specification
hyper docs create "Authentication Architecture" --type technical-spec

# User guide
hyper docs create "Getting Started Guide" --type user-guide

# API documentation
hyper docs create "REST API Reference" --type api-reference

# Process documentation
hyper docs create "Deployment Process" --type process

# Troubleshooting guide
hyper docs create "Common Issues" --type troubleshooting
```

## AI-Assisted Documentation

### Content Generation

```bash

  --type api-reference \
  --title "Authentication API"

  --type user-guide \
  --title "User Authentication Guide"

hyper docs generate --type troubleshooting \
  --source ./support-tickets \
  --title "Authentication Issues Guide"
```


## Documentation Templates

### Built-in Templates

```bash
# List available templates
hyper docs templates

# Show template structure
hyper docs template show technical-spec

# Create custom template
hyper docs template create my-spec-template
```

**Template Categories**:
- `technical-spec` - Technical specification template
- `user-guide` - User guide with screenshots and examples  
- `api-reference` - API documentation with OpenAPI structure
- `troubleshooting` - Problem-solution format
- `process` - Step-by-step process documentation
- `architecture` - System architecture with diagrams
- `security` - Security documentation template
- `onboarding` - New team member guide template

### Custom Templates

```yaml
# .hyper/doc-templates/my-api-template.yml
name: "API Endpoint Documentation"
type: "api-reference"

  - title: "Overview"

  - title: "Authentication"  
    required: true
  - title: "Endpoints"
    required: true
    subsections:
      - "Request Format"
      - "Response Format"
      - "Examples"
      - "Error Codes"
  - title: "Rate Limiting"
    required: false
  - title: "SDK Examples"
    required: false

frontmatter:
  - field: "api_version"
    required: true
  - field: "last_updated"
    auto_generate: true

examples:
  required: true
  languages: ["curl", "javascript", "python"]
```

## Team Collaboration

### Review Process

```bash
# Submit documentation for review
hyper docs review submit user-guide.md --reviewers alice,bob

# Review documentation changes
hyper docs review user-guide.md --approve

# Track review status
hyper docs review status
```

### Version Control

```bash
# Track documentation versions
hyper docs version user-guide.md --tag v1.2

# Show documentation history
hyper docs history user-guide.md

# Compare versions
hyper docs diff user-guide.md --versions v1.1,v1.2
```

## Integration with Development

### Code-Documentation Sync

```bash
# Update docs when code changes
hyper docs sync --auto-update

# Generate docs from code comments
hyper docs extract --source ./src --type api-reference

# Link documentation to code
hyper docs link user-guide.md --code-refs ./src/components
```

### Epic Integration

```bash
# Generate documentation for epic
hyper docs epic user-auth --generate-all

# List epic documentation
hyper docs list --epic user-auth

# Epic documentation completeness
hyper docs epic user-auth --coverage
```

## Configuration

### Documentation Settings

```javascript
// hyperdev.config.js
export default {
  docs: {
    // Documentation directory
    docs_dir: 'docs/',
    
    // Default settings
    defaults: {
      format: 'markdown',
      template: 'standard',
      ai_assist: false
    },
    
    // AI assistance
    ai: {
      enabled: true,
      provider: 'openai',
      model: 'gpt-4',
      features: ['generation', 'improvement', 'review']
    },
    
    // Quality standards
    quality: {
      min_readability_score: 7.0,
      max_age_days: 90,
      require_examples: true,
      require_reviews: true
    },
    
    // Publishing
    publishing: {
      site_generator: 'mintlify',
      theme: 'default',
      auto_deploy: true,
      export_formats: ['pdf', 'html']
    },
    
    // Integration
    integration: {
      sync_with_code: true,
      link_to_epics: true,
      auto_update_on_changes: true
    }
  }
};
```
