# hyper status

Comprehensive project health monitoring and status reporting command that provides real-time insights into project progress, code quality, epic status, team health, and system performance.

## Syntax

```bash
hyper status [options]
```

## Overview

The `hyper status` command provides a complete project overview:

- **Project health**: Overall project status and health metrics
- **Epic progress**: Status of all epics and their completion rates
- **Code quality**: Quality scores, test coverage, and technical debt
- **Security status**: Vulnerability assessments and security compliance
- **Performance metrics**: Build times, deployment status, and system performance  
- **Team health**: Developer productivity and collaboration metrics
- **System status**: Tool availability and configuration health

<Info>
  Think of `hyper status` as your project's dashboard - a single command that gives you
  complete visibility into your project's current state and health.
</Info>

## Options

| Option | Description | Default | Example |
|--------|-------------|---------|---------|
| `--health` | System health check | `false` | `--health` |
| `--quality` | Code quality metrics | `false` | `--quality` |
| `--security` | Security status | `false` | `--security` |
| `--progress` | Development progress | `false` | `--progress` |
| `--epics` | Epic status | `false` | `--epics` |
| `--team` | Team metrics | `false` | `--team` |
| `--performance` | Performance metrics | `false` | `--performance` |
| `--json` | JSON output format | `false` | `--json` |
| `--refresh` | Force refresh of status | `false` | `--refresh` |
| `--detailed` | Detailed status report | `false` | `--detailed` |

## Examples

### Basic Status

```bash
# Overall project status
hyper status

# Comprehensive status check  
hyper status --health --quality --security

# Epic-focused status
hyper status --epics --progress

# Team health check
hyper status --team --detailed
```

### Specific Status Checks

```bash
# Code quality only
hyper status --quality

# Security assessment
hyper status --security --detailed  

# Performance monitoring
hyper status --performance

# System health diagnostics
hyper status --health --verbose
```

## Status Export and Reporting

### JSON Export

```bash
# Export status as JSON
hyper status --json --output status-report.json

# Specific status export
hyper status --epics --json > epic-status.json
```

### Report Generation

```bash
# Generate comprehensive report
hyper status --report --detailed --output project-status.md

# Executive summary
hyper status --executive-summary --export pdf

# Team performance report
hyper status --team --report --period 1m
```

## Configuration

### Status Check Settings

```javascript
// hyperdev.config.js
export default {
  status: {
    // Default checks to run
    default_checks: [
      'health',
      'quality', 
      'security',
      'progress',
      'epics'
    ],
    
    // Health thresholds
    thresholds: {
      quality_score: 7.0,
      security_score: 8.0,
      performance_score: 7.5,
      team_health: 7.0,
      test_coverage: 75
    },
    
    // Alert configuration
    alerts: {
      enabled: true,
      channels: ['console', 'slack'],
      severity_levels: ['critical', 'high', 'medium']
    },
    
    // Refresh intervals
    cache: {
      status_cache_duration: 300,    // 5 minutes
      auto_refresh: true,
      background_updates: true
    }
  }
};
```

### Team Status Settings

```javascript
export default {

    },
    
    // Reporting schedule
    reports: {
      daily_standup: true,

    }
  }
};
```

## Integration Features

### Continuous Monitoring

```bash
# Set up continuous status monitoring
hyper status --monitor --interval 300  # Check every 5 minutes

# Alert on status changes
hyper status --alerts --webhook https://hooks.slack.com/...
```

### CI/CD Integration

```bash
# Status check in CI pipeline
hyper status --ci --exit-code  # Non-zero exit on issues

# Quality gate enforcement
hyper status --quality --gate --fail-on medium
```
