# Installation

> Deploy HyperDev in minutes and transform your development workflow

Get HyperDev installed and running on your system with our streamlined installation process. After completing this guide, you'll have a complete controlled AI development environment ready for immediate use.

<Info>
  **Prerequisites**: Before you begin, ensure you have [Node.js v18+](https://nodejs.org/) and [Bun](https://bun.sh/) installed on your system.
</Info>

## System Requirements

Before installing HyperDev, verify your system meets these requirements:

<Card title="Operating System Support" icon="desktop">
  **Supported Platforms:**
  - macOS 10.15 (Catalina) or later
  - Ubuntu 20.04+ or equivalent Linux distribution
  - Windows 11 with WSL2 (recommended) or Windows 10 build 19041+
</Card>

<Card title="Runtime Requirements" icon="cog">
  **Required Software:**
  - **Node.js**: Version 18.0.0 or higher
  - **Package Manager**: Bun (recommended) or npm/yarn
  - **Git**: Latest stable version
  - **Terminal**: Modern terminal with Unicode support
</Card>

The fastest way to get started with HyperDev is our one-command installer:

<CodeGroup>

  ```bash Windows (PowerShell)
  Invoke-WebRequest -Uri https://install.hyperdev.ai/windows.ps1 | Invoke-Expression
  ```


## Manual Installation

For advanced users who prefer granular control over the installation process:

<AccordionGroup>
  <Accordion title="Step 1: Install Core CLI">
    Install the HyperDev command-line interface:

    <CodeGroup>
      ```bash Bun (Recommended)
      bun install -g @hyperdev/cli
      ```

      ```bash npm
      npm install -g @hyperdev/cli
      ```

      ```bash yarn
      yarn global add @hyperdev/cli
      ```
    </CodeGroup>

    <Tip>
      Bun provides significantly faster installation and better performance. Install Bun first if you haven't already.
    </Tip>
  </Accordion>

  <Accordion title="Step 2: Install Tool Suite">
    Install the complete HyperDev toolkit:

    ```bash
    # Install all tools at once
    hyper install --all

    # Or install individually
    hyper install gen epics dev dash
    ```

    Each tool provides specialized functionality:
    - **gen**: AI-augmented code generation
    - **epics**: Strategic planning and architecture
    - **dev**: Execution control and monitoring
    - **dash**: Performance dashboard and analytics
  </Accordion>

  <Accordion title="Step 3: Initialize Environment">
    Set up your development environment:

    ```bash
    # Initialize HyperDev in current directory
    hyper init

    # Configure development monitoring
    hyper dev init --hooks enabled

    # Verify installation
    hyper doctor
    ```
  </Accordion>
</AccordionGroup>

## Verification & Testing

Verify your installation is working correctly:

<Steps>
  <Step title="Run System Check">
    Execute the built-in diagnostic tool:
    
    ```bash
    hyper doctor
    ```
    
    This checks all dependencies, configurations, and tool availability.
  </Step>
  
  <Step title="Test Code Generation">
    Try generating a simple project:
    
    ```bash
    hyper gen quick-start --template react-typescript
    ```
  </Step>
  
  <Step title="Verify Tool Integration">
    Ensure all tools are properly linked:
    
    ```bash
    hyper --version
    hyper gen --version
    hyper epics --version
    hyper dev --version
    hyper dash --version
    ```
  </Step>
</Steps>

## Authentication & Configuration

### Account Setup

<Warning>
  You need a HyperDev account to access advanced features and cloud synchronization. [Create your free account](https://dashboard.hyperdev.ai/signup) if you haven't already.
</Warning>

Authenticate with your HyperDev account:

```bash
hyper auth login
```

This opens your browser for OAuth authentication or you can use API tokens:

```bash
hyper auth token --set YOUR_API_TOKEN
```

### Workspace Configuration

Initialize your development workspace:

<CodeGroup>
  ```bash Initialize Current Directory
  hyper workspace init
  ```

  ```bash Create New Workspace
  hyper workspace create my-awesome-project
  cd my-awesome-project
  ```

  ```bash Clone Template Workspace
  hyper workspace clone --template fullstack-saas
  ```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Permission errors on macOS/Linux">
    Fix file permissions for HyperDev configuration:
    
    ```bash
    sudo chown -R $(whoami) ~/.hyperdev
    chmod -R 755 ~/.hyperdev
    ```
    
    Or install without sudo:
    ```bash
    npm config set prefix '~/.local'
    echo 'export PATH=$HOME/.local/bin:$PATH' >> ~/.bashrc
    ```
  </Accordion>

  <Accordion title="Node.js version conflicts">
    Use a Node.js version manager to handle multiple versions:
    
    <CodeGroup>
      ```bash nvm
      nvm install 18
      nvm use 18
      nvm alias default 18
      ```
      
      ```bash fnm (faster alternative)
      fnm install 18
      fnm use 18
      fnm default 18
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Installation hanging or failing">
    Try these solutions in order:
    
    1. **Clear package cache:**
       ```bash
       bun cache clear
       npm cache clean --force
       ```
    
    2. **Use different registry:**
       ```bash
       bun install -g @hyperdev/cli --registry https://registry.npmjs.org/
       ```
    
    3. **Install with verbose logging:**
       ```bash
       bun install -g @hyperdev/cli --verbose
       ```
  </Accordion>

  <Accordion title="Windows WSL2 specific issues">
    Common Windows/WSL2 solutions:
    
    1. **Update WSL2:**
       ```bash
       wsl --update
       ```
    
    2. **Fix line ending issues:**
       ```bash
       git config --global core.autocrlf false
       ```
    
    3. **Set proper WSL2 path:**
       ```bash
       echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.bashrc
       ```
  </Accordion>
</AccordionGroup>

## Updating HyperDev

Keep your HyperDev installation current with the latest features and security updates:

<CodeGroup>
  ```bash Update All Tools
  hyper update
  ```

  ```bash Update Specific Tool
  hyper update gen
  ```

  ```bash Check for Updates
  hyper update --check
  ```
</CodeGroup>

<Tip>
  HyperDev automatically checks for updates weekly and can be configured to update automatically with `hyper config set auto-update true`.
</Tip>

The update process handles:
- Dependency compatibility checking
- Configuration migration
- IDE extension updates
- Template repository synchronization
