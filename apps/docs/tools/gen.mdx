# gen: AI-Augmented Code Generation

The **gen** tool is HyperDev's AI-augmented template-based code generation system. It combines the power of traditional templates with AI intelligence to create secure, consistent, and maintainable code at unprecedented speed.

## Core Capabilities

### Template-Based Generation
- **Intelligent Template Discovery**: Multi-source template discovery from npm, GitHub, and local directories
- **Trust System**: Security validation and trust scoring for external templates
- **Composition Framework**: Systematic combination of multiple templates
- **Pattern Enforcement**: Consistent code generation following established patterns

### AI Augmentation
- **Context-Aware Generation**: AI understands project context and requirements
- **Dynamic Template Adaptation**: Templates adapt to specific project needs
- **Quality Enhancement**: AI improves generated code quality and consistency
- **Pattern Recognition**: Learns from existing codebase patterns

## Subcommands

### `hyperdev gen <template>`
Generate code from a specified template with AI augmentation.

```bash
# Generate from local template
hyperdev gen react-component

# Generate from npm package
hyperdev gen npm:@company/api-template

# Generate from GitHub repository
hyperdev gen github:username/template-repo

# Generate with AI context enhancement
hyperdev gen api-endpoint --ai-context="REST API for user management"
```

### `hyperdev gen search <query>`
Intelligent template discovery with AI-powered matching.

```bash
# Search for React templates
hyperdev gen search "react component"

# Find API templates with specific features
hyperdev gen search "express api with authentication"

# Discover templates by technology stack
hyperdev gen search "typescript next.js tailwind"
```

### `hyperdev gen list`
List available templates with trust scores and metadata.

```bash
# List all available templates
hyperdev gen list

# Filter by trust level
hyperdev gen list --trusted-only

# Show template details
hyperdev gen list --detailed
```

### `hyperdev gen pack <name>`
Package templates for distribution with validation.

```bash
# Package current template
hyperdev gen pack my-template

# Package with validation
hyperdev gen pack my-template --validate

# Package for npm distribution
hyperdev gen pack my-template --npm-ready
```

### `hyperdev gen validate`
Validate template structure and security.

```bash
# Validate current template
hyperdev gen validate

# Validate specific template
hyperdev gen validate path/to/template

# Deep security validation
hyperdev gen validate --security-scan
```

## Template Architecture

### Single Template Structure
```
my-template/
├── template.yml          # Template metadata and configuration
├── _templates/           # Template files directory
│   ├── component.tsx.ejs.t   # EJS template file
│   ├── test.spec.ts.ejs.t    # Test template
│   └── styles.css.ejs.t      # Style template
└── action.ts            # Optional: Custom generation actions (V8)
```

### Multi-Template Structure
```
template-collection/
├── template.yml          # Collection metadata
├── templates/           # Multiple template definitions
│   ├── component/
│   │   ├── template.yml
│   │   └── _templates/
│   └── service/
│       ├── template.yml
│       └── _templates/
└── shared/              # Shared template assets
```

### Template Configuration (`template.yml`)
```yaml
name: "React Component Template"
description: "Generate React components with TypeScript and tests"
version: "1.0.0"
trust_score: 9.2

# AI Enhancement Context
ai_context:
  framework: "React 18"
  language: "TypeScript"
  patterns: ["hooks", "functional-components"]
  
# Template Variables
variables:
  - name: "componentName"
    type: "string"
    required: true
    description: "Name of the React component"
  - name: "withTests" 
    type: "boolean"
    default: true
    description: "Include test files"

# Security Configuration
security:
  trusted_sources: ["npm:@company/*", "github:org/*"]
  scan_generated: true
  validate_dependencies: true

# Quality Controls
quality:
  enforce_patterns: true
  validate_syntax: true
  check_best_practices: true
```

## Trust System

### Trust Score Calculation
The trust system evaluates templates based on multiple factors:

- **Source Reputation**: Known and verified template sources (8.0-10.0)
- **Security Validation**: Automated security scanning results (7.0-9.0)
- **Community Usage**: Download counts and user feedback (6.0-8.0)
- **Code Quality**: Automated quality assessment (7.0-9.0)
- **Maintenance**: Regular updates and issue resolution (6.0-8.0)

### Trust Levels
- **10.0 - Verified**: HyperDev official templates
- **8.0-9.9 - Trusted**: Well-established community templates
- **6.0-7.9 - Reviewed**: Community templates with basic validation
- **4.0-5.9 - Caution**: Use with additional review
- **0.0-3.9 - Untrusted**: Requires manual security validation

### Security Validation Pipeline
```yaml
security_pipeline:
  static_analysis:
    - tool: semgrep
      rules: owasp-top-10
    - tool: bandit
      config: strict
      
  dependency_analysis:
    - tool: npm-audit
    - tool: snyk
    
  secrets_detection:
    - tool: truffleHog
    - tool: detect-secrets
    
  template_validation:
    - check: no_hardcoded_secrets
    - check: safe_file_operations
    - check: input_sanitization
```

## Advanced Features

### Template Composition
Combine multiple templates for complex scenarios:

```bash
# Compose API and frontend templates
hyperdev gen compose api-backend+react-frontend

# Layer templates with inheritance
hyperdev gen base-template --extend=auth-layer,api-layer
```

### Custom Actions (V8 Feature)
Extend templates with TypeScript actions:

```typescript
// action.ts
export class ComponentAction {
  @validate({ required: true })
  async generateComponent(context: GenerationContext) {
    // Custom generation logic with AI integration
    const aiResponse = await this.aiService.enhance(context);
    return this.renderWithAI(aiResponse);
  }
}
```

### Performance Optimization
- **Lazy Loading**: Templates load on-demand for faster startup
- **Caching**: Multi-level caching for template discovery and generation
- **Parallel Processing**: Concurrent template processing for complex compositions
- **Hash-Based Storage**: Efficient template indexing and retrieval

## Troubleshooting

### Common Issues
1. **Template Not Found**: Check template name and source availability
2. **Trust Score Too Low**: Use `--force` flag or find higher-trust alternative
3. **Generation Errors**: Validate template variables and context
4. **AI Enhancement Failures**: Check AI service availability and context

### Debug Commands
```bash
# Enable verbose logging
hyperdev gen --verbose template-name

# Show template resolution process
hyperdev gen --trace-resolution template-name

# Validate template without generation
hyperdev gen --dry-run template-name
```
