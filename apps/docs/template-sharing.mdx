### Package Configuration

Configure your template package for optimal distribution:

```javascript
// hyperdev-package.config.js
module.exports = {
  name: '@myorg/react-templates',
  description: 'Professional React component templates with TypeScript and testing support',
  version: '2.1.0',
  author: 'Your Organization <dev@yourorg.com>',
  
  // Template registry - maps names to directories
  templates: {
    'component': './templates/component',
    'hook': './templates/hook', 
    'service': './templates/service',
    'page': './templates/page',
    'workflow': './templates/workflow'
  },
  
  // Dependencies required by generated code
  peerDependencies: {
    'react': '^18.0.0',
    'typescript': '^4.9.0',
    '@types/react': '^18.0.0'
  },
  
  // Development dependencies for template actions
  devDependencies: {
    '@hyperdev/action-system': '^2.0.0',
    'jest': '^29.0.0'
  },
  
  // Template metadata for discovery
  keywords: ['react', 'typescript', 'components', 'hooks'],
  category: 'frontend',
  license: 'MIT',
  homepage: 'https://github.com/myorg/react-templates',
  
  // Distribution settings
  publishConfig: {
    registry: 'https://registry.npmjs.org',
    access: 'public'
  },
  
  // Quality requirements
  engines: {
    node: '>=16.0.0',
    hyperdev: '^8.0.0'
  },
  
  // Trust and security settings
  trusted: true,
  maintainers: [
    'dev@yourorg.com',
    'security@yourorg.com'
  ]
};
```

## Chapter 2: Publishing to Package Registries

### NPM Registry Publishing

Publish to NPM for maximum discoverability:

<Steps>
  <Step title="Prepare Package">
    Validate and prepare your template package:
    
    ```bash
    # Validate package structure and metadata
    bunx hyper validate-package
    
    # Test all templates
    bun run test
    
    # Build optimized package
    bun run build
    ```
  </Step>
  
  <Step title="Version Management">
    Use semantic versioning for clear upgrade paths:
    
    ```bash
    # Patch release (bug fixes)
    bun version patch
    
    # Minor release (new features)
    bun version minor
    
    # Major release (breaking changes)
    bun version major
    ```
  </Step>
  
  <Step title="Publish">
    Publish to NPM registry:
    
    ```bash
    # Public package
    bun publish
    
    # Scoped package
    bun publish --access public
    
    # With distribution tag
    bun publish --tag beta
    ```
  </Step>
</Steps>

### Private Registry Setup

Configure private registries for enterprise distribution:

```bash
# Configure private registry

# Set authentication

# Publish to private registry
```

### Automated Repository Publishing

Set up automated publishing with GitHub Actions:

```yaml
# .github/workflows/release.yml
name: Release Templates

on:
  push:
    tags: ['v*']

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v1
      
      - name: Install dependencies
        run: bun install
        
      - name: Validate templates
        run: bunx hyper validate-package
        
      - name: Run tests
        run: bun test
        
  publish:
    needs: validate
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v1
      
      - name: Configure NPM
        run: |
          echo "//registry.npmjs.org/:_authToken=${{ secrets.NPM_TOKEN }}" > ~/.npmrc
          
      - name: Publish to NPM
        run: bun publish
        
      - name: Update marketplace
        run: bunx hyper marketplace update
        env:
          HYPERGEN_API_KEY: ${{ secrets.HYPERGEN_API_KEY }}
        
      - name: Create GitHub Release
        uses: actions/create-release@v1
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          tag_name: ${{ github.ref }}
          release_name: Release ${{ github.ref }}
```

## Troubleshooting Distribution Issues

### Common Problems and Solutions

<AccordionGroup>
  <Accordion title="Template not found after installation">
    **Symptoms**: `hyper gen` reports template not found despite successful installation
    
    **Solutions**:
    ```bash
    # Refresh template cache
    bunx hyper cache clear
    
    # Verify installation
    bunx hyper list --verbose
    
    # Check discovery paths
    bunx hyper config get discovery.paths
    
    # Force re-discovery
    bunx hyper refresh --all
    ```
  </Accordion>

  <Accordion title="Version conflicts between templates">
    **Symptoms**: Conflicting dependencies or incompatible template versions
    
    **Solutions**:
    ```bash
    # Check dependency tree
    bunx hyper deps --tree
    
    # Resolve conflicts
    bunx hyper resolve-conflicts
    
    # Pin specific versions
    bunx hyper pin @org/template 1.2.3
    
    # Use resolution overrides
    bunx hyper config set resolutions.@org/template "1.2.3"
    ```
  </Accordion>

  <Accordion title="Private registry authentication issues">
    **Symptoms**: Authentication failures when accessing private templates
    
    **Solutions**:
    ```bash
    # Re-authenticate
    bunx hyper auth login --registry https://private-registry.com
    
    # Check token validity
    bunx hyper auth status
    
    # Update credentials
    bunx hyper auth refresh
    
    # Clear auth cache
    bunx hyper auth logout --all
    ```
