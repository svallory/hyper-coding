# Tutorial Validation Guide

> **Companion guide to the Getting Started Tutorial**  
> Use this guide to validate your progress and troubleshoot issues at each step.

This validation guide provides specific commands and expected outputs for each section of the [Getting Started Tutorial](/getting-started-tutorial). Use it to verify your setup is working correctly and troubleshoot any issues.

## Part 1: Installation & Environment Verification

### 1.1 Installation Validation

<Steps>
  <Step title="Verify Installation">
    ```bash
    # Expected: Version number like "8.x.x"
    hyper --version
    ```
    
    **‚úÖ Success Indicators:**
    - Version number displays (e.g., "8.2.1")
    - No "command not found" errors
    
    **‚ùå Troubleshooting:**
    - **Command not found**: Install with `bun install -g hyperdev`
    - **Permission errors**: Use `bun install -g hyperdev --user`
  </Step>
  
  <Step title="System Status Check">
    ```bash
    # Expected: Status report with configuration details
    hyper system status
    ```
    
    **‚úÖ Success Indicators:**
    - System status displays without errors
    - Shows cookbook paths and configuration
    
    **‚ùå Troubleshooting:**
    - **No configuration found**: This is normal for first run
    - **Permission errors**: Check file permissions in home directory
  </Step>
</Steps>

### 1.2 Environment Validation

<AccordionGroup>
<Accordion title="Node.js Version Check">

```bash
# Must be 18+ for compatibility
node --version

# Alternative check with more details
node -p "process.version"
```

**Expected Output Format:** `v18.x.x` or higher

**Common Issues:**
- **v16.x.x or lower**: Upgrade Node.js to 18+
- **No node command**: Install Node.js from [nodejs.org](https://nodejs.org)

</Accordion>

<Accordion title="Recipe Discovery Validation">

```bash
# Should show built-in recipes
hyper gen list

# More detailed information
hyper gen discover --verbose
```

**Expected Results:**
- At least 1-3 built-in recipes listed
- No error messages about cookbook access

**Troubleshooting:**
- **Empty list**: Check internet connection for remote cookbook access
- **Permission errors**: Ensure write access to `~/.hyperdev/` directory

</Accordion>
</AccordionGroup>

## Common Issues & Solutions

### Installation Issues

<AccordionGroup>
<Accordion title="Command not found: hyper">

**Diagnostic:**
```bash
which hyper
echo $PATH
bun list -g | grep hyperdev
```

**Solutions:**
1. **Install globally:** `bun install -g hyperdev`
2. **Check PATH:** Ensure bun global bin directory is in PATH
3. **Use bunx:** `bunx hyper` as alternative

</Accordion>

<Accordion title="Permission Denied Errors">

**Diagnostic:**
```bash
ls -la $(which hyper 2>/dev/null || echo "/usr/local/bin/hyper")
whoami
groups
```

**Solutions:**
1. **Fix permissions:** `chmod +x $(which hyper)`
2. **Use user install:** `bun install -g hyperdev --user`
3. **Check ownership:** `sudo chown $USER:$USER ~/.bun`

</Accordion>
</AccordionGroup>

### Generation Issues

<AccordionGroup>
<Accordion title="Recipe not found">

**Diagnostic:**
```bash
hyper gen list
hyper gen discover --verbose
ls -la cookbooks/
```

**Solutions:**
1. **Run discovery:** `hyper gen discover`
2. **Check paths:** Verify cookbook directory structure
3. **Use full paths:** `hyper gen ./cookbooks/my-recipe/recipe.yml`

</Accordion>

<Accordion title="Template syntax errors">

**Diagnostic:**
```bash
hyper gen validate path/to/recipe.yml
head -20 path/to/template.liquid
```

**Solutions:**
1. **Check Liquid syntax:** Ensure `{{ }}` and `{% %}` are correct
2. **Validate YAML:** Check recipe.yml frontmatter
3. **Test variables:** Use `--dry-run` to preview output

</Accordion>
</AccordionGroup>

### File System Issues

<AccordionGroup>
<Accordion title="Files not generated">

**Diagnostic:**
```bash
ls -la target/directory/
pwd
DEBUG=hyperdev* hyper gen path/to/recipe.yml --name=Test
```

**Solutions:**
1. **Check permissions:** Ensure write access to target directory
2. **Verify paths:** Confirm current working directory is correct
3. **Use absolute paths:** Specify full paths in templates

</Accordion>

<Accordion title="Overwrite conflicts">

**Diagnostic:**
```bash
ls -la src/components/ExistingComponent/
hyper gen path/to/recipe.yml --name=ExistingComponent --dry-run
```

**Solutions:**
1. **Use force flag:** Add `--force` to overwrite
2. **Remove existing:** Delete existing files first
3. **Different name:** Use a different component name

</Accordion>
</AccordionGroup>
