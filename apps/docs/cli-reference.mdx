# HyperDev CLI Reference

The HyperDev CLI is the command-line interface for the scalable code generator that saves you time. This reference provides comprehensive documentation for all commands, options, and workflows.

<Info>
  This is the definitive reference for the `hyper` command-line tool. For conceptual overviews, see the [CLI Overview](/cli/overview).
</Info>

## Installation & Setup

### Installing HyperDev CLI

```bash
# Install via npm/bun
bun install -g hyper
# or
npm install -g hyper

# Verify installation
hyper system version
```

### First Time Setup

Initialize your first workspace to get started:

```bash
# Initialize a generator workspace with examples
hyper init workspace --withExamples=true

# Create your first generator
hyper init generator --name=my-component --framework=react

# Discover available generators
hyper discover

# List available actions
hyper list
```

## CLI Design Philosophy

### Progressive Complexity
- **Basic**: Simple action execution and template discovery
- **Intermediate**: Recipe system for complex workflows
- **Advanced**: Configuration, custom generators, and power features

### Intelligent Discovery
Commands automatically discover generators and provide helpful suggestions:
```bash
hyper gen my-component  # Auto-discovers if not found
hyper list component    # Filters by category
hyper info button       # Shows detailed parameter info
```

### Safety First
Built-in safety features prevent accidental file overwrites:
```bash
hyper gen create-api --dryRun      # Preview changes
hyper gen create-api --force       # Override safety checks
```

## Core Commands

### Action Management

#### `hyper gen <name> [parameters...] [options]`

Execute a code generation action with optional parameters. This is the primary command for code generation.

**Syntax:**
```bash
hyper gen <action-name|recipe-file> [--param=value] [--flag]
```

**Smart Action Resolution:**
HyperDev intelligently resolves actions and recipes:
- Registered actions by name: `hyper gen create-component`
- Recipe files directly: `hyper gen my-component.yml`
- Auto-discovery: Searches for `_recipes/<name>.yml` if action not found

**Options:**
- `--defaults` - Use default values instead of prompting
- `--dryRun` - Preview changes without writing files
- `--force` - Force overwrite existing files
- `--<param>=<value>` - Set action/recipe parameters

**Examples:**

<Tabs>
  <Tab title="Action Execution">
```bash
# Execute with parameters
hyper gen create-component --name=Button --type=tsx

# Use defaults for quick generation
hyper gen create-api --name=UserAPI --defaults

# Preview changes before applying
hyper gen create-component --name=Header --dryRun

# Force overwrite existing files
hyper gen update-schema --name=User --force
```
  </Tab>
  <Tab title="Recipe File Execution">
```bash
# Execute recipe file directly
hyper gen my-component.yml --name=Button --typescript=true

# Execute with dry run
hyper gen recipes/react-component.yml --name=Header --dryRun

# Auto-discovery of recipe files
hyper gen fullstack-app --name=MyApp --database=postgresql
# Searches for: fullstack-app.yml, _recipes/fullstack-app.yml
```
  </Tab>
  <Tab title="Parameter Formats">
```bash
# String parameters
--name=MyComponent --path=src/components

# Boolean parameters  
--typescript=true --tests=false

# JSON parameters for complex values
--config='{"strict": true, "target": "es2020"}'
--options='["async", "export"]'

# Space-separated format
hyper gen create-api --name UserAPI --method GET
```
  </Tab>
</Tabs>

Discover generators from all configured sources or specific sources.

**Sources:**
- `local` - Local generators in `recipes/` directory
- `npm` - Installed npm packages with generators

**Examples:**

### Recipe System (V8 Features)

<Callout type="note">
  **New in V8**: The Recipe System is HyperDev's most powerful feature, enabling multi-step code generation workflows with dependency management, progress tracking, and advanced error handling.
</Callout>

The Recipe System allows you to define complex, multi-step code generation workflows using YAML configuration files. Each recipe can contain multiple steps using different tools (template, action, codemod, recipe).

#### `hyper recipe execute <recipe> [options...]`

Execute a complete recipe with all its steps, including real-time progress tracking and sophisticated error handling.

**Syntax:**
```bash
hyper recipe execute <recipe.yml> [--var=value] [--flag]
```

**Advanced Options:**
- `--dryRun` - Preview all changes without writing files
- `--force` - Force overwrite existing files
- `--skipPrompts` - Use defaults, skip interactive prompts  
- `--continueOnError` - Continue execution after step failures
- `--<variable>=<value>` - Set recipe variables

**Real-time Progress Tracking:**
During execution, you'll see live progress updates:
```bash
üìà setup-database: initializing (10%)
‚úÖ Step: setup-database (1250ms)
üìà create-api: generating (45%)
‚úÖ Step: create-api (2100ms)
üìà create-frontend: processing (78%)
‚úÖ Step: create-frontend (1800ms)
```

**Examples:**

<Tabs>
  <Tab title="Basic Execution">
```bash
# Execute a recipe with variables
hyper recipe execute fullstack-app.yml --name=MyApp --database=postgresql

# Preview recipe execution
hyper recipe execute api-service.yml --name=UserService --dryRun

# Execute with error handling
hyper recipe execute complex-setup.yml --continueOnError --force
```
  </Tab>
  <Tab title="Advanced Execution">
```bash
# Execute with complex variables
hyper recipe execute microservice.yml \
  --name=AuthService \
  --features='["jwt", "oauth", "2fa"]' \
  --env=production \
  --continueOnError

# Execute with progress callbacks
hyper recipe execute deployment.yml \
  --environment=staging \
  --skipPrompts \
  --force
```
  </Tab>
  <Tab title="Error Recovery">
```bash
# Continue execution after failures
hyper recipe execute migration.yml --continueOnError

# Force execution with file overwrites
hyper recipe execute setup.yml --force --skipPrompts
```
  </Tab>
</Tabs>

#### `hyper step execute <recipe> <step-name> [options...]`

Execute a single step from a recipe with full isolation and dependency checking.

**Syntax:**
```bash
hyper step execute <recipe-path> <step-name> [--var=value] [--flag]
```

**Options:**
- `--dryRun` - Preview step execution without making changes
- `--force` - Force step execution even with dependency issues
- `--<variable>=<value>` - Set step variables

**Examples:**

<Tabs>
  <Tab title="Basic Step Execution">
```bash
# Execute specific step
hyper step execute fullstack-app.yml setup-database --database=postgresql

# Preview step execution
hyper step execute api-service.yml create-endpoints --dryRun

# Execute with variables
hyper step execute component.yml generate-component --name=Button --typescript=true
```
  </Tab>
  <Tab title="Advanced Step Execution">
```bash
# Execute step with complex variables
hyper step execute microservice.yml generate-api \
  --name=UserService \
  --endpoints='["users", "auth", "profile"]' \
  --dryRun

# Force step execution (skip dependency checks)
hyper step execute complex-recipe.yml build-step --force

# Execute with debug information
hyper step execute workflow.yml validation-step --debug
```
  </Tab>
</Tabs>

**Step Execution Results:**

### Project Initialization

#### `hyper init generator [options...]`

Create a new generator with scaffolded structure.

**Options:**
- `--name=<name>` - Generator name (required)
- `--description=<desc>` - Generator description
- `--category=<category>` - Generator category [default: custom]
- `--author=<author>` - Author name
- `--directory=<dir>` - Output directory [default: recipes]
- `--type=<type>` - Generator type: action, template, both [default: both]
- `--framework=<framework>` - Target framework: react, vue, node, cli, api, generic [default: generic]
- `--withExamples=<boolean>` - Include example templates [default: true]
- `--withTests=<boolean>` - Include test templates [default: true]

**Examples:**
```bash
# Create a basic generator
hyper init generator --name=my-widget

# Create a React component generator
hyper init generator --name=react-component --framework=react --category=component

# Create an API generator without examples
hyper init generator --name=api-endpoint --framework=node --withExamples=false
```

#### `hyper init workspace [options...]`

Initialize a generator workspace with example generators.

**Options:**
- `--directory=<dir>` - Workspace directory [default: recipes]
- `--withExamples=<boolean>` - Include example generators [default: true]

**Examples:**
```bash
# Initialize workspace with examples
hyper init workspace

# Initialize clean workspace
hyper init workspace --withExamples=false --directory=generators
```

### Error Handling & Recovery

```bash

  hyper recipe execute my-recipe.yml --name=MyProject

  echo "Recipe validation failed"
  exit 1
fi

# Dry run for safety
hyper gen risky-operation --dryRun || {
  echo "Dry run revealed issues, aborting"
  exit 1
}
```

## Troubleshooting

### Common Issues

**Action Not Found**
```bash
$ hyper gen my-component
‚ùå Action 'my-component' not found

# Solutions:
hyper discover              # Rediscover generators
hyper list                  # See available actions
hyper list component        # Filter by category
```

**Template Validation Errors**
```bash
$ hyper template validate template.yml
‚ùå Template validation failed:
  ‚Ä¢ Missing required field: name
  ‚Ä¢ Invalid variable type: config.timeout

# Solutions:
# 1. Fix template.yml syntax
# 2. Check variable definitions
# 3. Validate against schema
```

**Recipe Execution Failures**
```bash
$ hyper recipe execute broken-recipe.yml
‚ùå Recipe execution failed: Step 2 failed

# Solutions:
hyper recipe validate broken-recipe.yml  # Check syntax
hyper step list broken-recipe.yml        # Review steps
hyper recipe execute broken-recipe.yml --continueOnError  # Skip failures
```

### Naming Conventions

- **Actions**: `verb-noun` (create-component, update-schema)  
- **Templates**: `technology-type` (react-component, express-api)
- **Recipes**: `descriptive-name` (fullstack-app, microservice-setup)
- **Parameters**: `camelCase` (componentName, apiEndpoint)

### Error Handling

Always validate inputs and provide recovery options:

