# Security Best Practices

Basic security guidelines for safe template usage and code generation with HyperDev.

## Basic Security Guidelines

### Template Sources

Be careful about template sources:

```javascript
// hyperdev.config.js - Basic trust configuration
export default {
  // Trust local and verified sources
  trust: {
    'github:company/*': 'high',
    '@company/cookbook-*': 'high'
  }
};
```

### Basic Validation

HyperDev includes basic security features:
- Templates are validated before use
- File paths are checked for safety
- Basic secret detection

### Template Development

### Basic Security Checklist

When using templates:

- ✅ Review template source and author
- ✅ Test templates with `--dry-run` first
- ✅ Check generated output before using
- ✅ Keep templates updated
- ✅ Use trusted sources when possible
- ✅ Avoid templates that require elevated permissions

### Common Security Tips

**Template Sources:**
- Prefer official and community-verified sources
- Review template code before first use
- Be cautious with templates from unknown sources

**File Generation:**
- Always review generated files
- Check file paths for safety
- Use version control to track changes

**Development Workflow:**
- Test templates in isolated environments
- Use `--dry-run` to preview changes
- Keep backups of important files

This covers the basic security practices for safe HyperDev usage.
